Configuração de serviços eficaz e rápida em Go

January 1, 12 min read

Por quê?

É muito vantajoso ser capaz de fácilmente configurar serviços em produção sem realizar alterações de código.

Mas quais tipos de configuração? os parâmetros mais comuns a serem expostos são:

  • Endereços de API
  • Caminho de arquivos e diretórios do sistema operacional
  • Nível de log (Error, Warning, Info, Debug)
  • Timeouts de clientes HTTP e banco e dados
  • TTL de cache (Time to Live)
  • Número de workers

Tendo em vista que nem todas as equipes possuem um processo de release simples, em que basta você fazer um commit para alterar o que está em produção, é bastante interessante ter estas configurações “expostas” e não “hard-coded”, pois em momentos críticos podem ser crúciais na mitigação de incidentes.

Usando envconfig

envconfig é uma biblioteca criada por Kelsey Hightower para simplificar o carregamento de variáveis de ambiente, observe o exemplo a seguir.

package main

import (
	"log"
	"time"

	"github.com/kelseyhightower/envconfig"
)

const appName = "EXEMPLO"

type Config struct {
	LogLevel     string        `default:"info"`
	Timeout      time.Duration `default:"30s"`
	Workers      int           `required:"true"`
	InstagramAPI string        `default: "instagram.com/api"`
}

func main() {
	config := &Config{}

	err := envconfig.Process(appName, config)
	if err != nil {
		log.Fatalln("Erro ao carregar configuração", err)
	}

	log.Printf("Iniciando a aplicação com as configurações\n%+v", config)
}

E então você pode configurar sua aplicação usando variáveis de ambiente

EXEMPLO_WORKERS=10 EXEMPLO_TIMEOUT="40s" go run main.go

Exemplo utilizando ConfigMaps no Kubernetes

Você pode definir um configmap com todas as variáveis de ambiente que você deseja configurar.

Se você usar o exemplo de código acima seus Pods irão ficar em CrashLoop, pois o programa irá mostrar as configurações e finalizar. complemente o exemplo acima com um serviço backend real. Os manifests abaixo são apenas para lhe guiar em como carregar o configmap inteiro em variáveis de ambiente.

apiVersion: v1
kind: ConfigMap
metadata:
  name: exemplo-config
data:
  EXEMPLO_WORKERS: "3"
  EXEMPLO_TIMEOUT: "40s"
  EXEMPLO_LOG_LEVEL: "debug"

E Então na especificação de variáveis de ambiente do seu pod:

envFrom:
  - configMapRef:
    name: exemplo-config

Exemplo completo:

apiVersion: v1
kind: ConfigMap
metadata:
  name: exemplo-config
data:
  EXEMPLO_WORKERS: "3"
  EXEMPLO_TIMEOUT: "40s"
  EXEMPLO_LOG_LEVEL: "debug"
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: exemplo-service
  labels:
    app: exemplo
spec:
  replicas: 1
  selector:
    matchLabels:
      app: exemplo
  template:
    metadata:
      labels:
        app: exemplo
    spec:
      containers:
      - name: exemplo-container
        image: exemplo-service:latest
        imagePullPolicy: Never
        envFrom:
          - configMapRef:
              name: "exemplo-config" # Referência do configmap
        ports:
        - containerPort: 80
